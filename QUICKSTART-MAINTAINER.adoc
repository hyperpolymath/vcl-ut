= vcl-total — Quick Start for Platform Maintainers
:toc:
:toclevels: 2

== Overview

This guide covers packaging, deploying, and maintaining vcl-total for
distribution on your platform.

== Runtime Dependencies

idris2, zig

== Build from Source

[source,bash]
----
git clone https://github.com/hyperpolymath/vql-ut.git
cd vcl-total
just build-release
----

Output: `{{BUILD_OUTPUT_PATH}}`

== Packaging

=== Guix

[source,bash]
----
guix build -f guix.scm
----

=== Nix

[source,bash]
----
nix build
----

=== Container (Stapeln)

[source,bash]
----
just stapeln-export    # Generates Containerfile
podman build -t vcl-total .
----

=== Manual Package

[source,bash]
----
just install --prefix=/usr/local
----

Files installed:

[cols="1,2"]
|===
| Path | Contents

| `$PREFIX/bin/`
| Executables

| `$PREFIX/share/vcl-total/`
| Data files, assets

| `$PREFIX/share/doc/vcl-total/`
| Documentation

| `$PREFIX/share/applications/`
| .desktop file (Linux, if GUI)

| `$PREFIX/share/man/man1/`
| Man pages
|===

== Configuration

Default config location: `$XDG_CONFIG_HOME/vcl-total/config.toml`

Fallback: `$HOME/.config/vcl-total/config.toml`

== Health Checks

[source,bash]
----
just doctor            # Full diagnostic
just run --version   # Version check
just run --selftest  # Built-in self-test
----

== Updating

[source,bash]
----
git pull
just build-release
just install --prefix=/usr/local
----

Or via OPSM: `opsm update vcl-total`

== Multi-Instance Deployment

For deploying multiple instances (e.g., different users or tenants):

[source,bash]
----
just install --prefix=/opt/vcl-total-instance1 --config=/etc/vcl-total/instance1.toml
just install --prefix=/opt/vcl-total-instance2 --config=/etc/vcl-total/instance2.toml
----

Each instance has isolated config, data, and logs.
